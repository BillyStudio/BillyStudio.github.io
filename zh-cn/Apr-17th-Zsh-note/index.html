Shell Programming | Zsh

4月 17 2019 Linux shell 3 分钟 读完 (约 425 字)

Zsh 是一个很棒的 shell，但需要很多前置的配置工作才能最大发挥它的价值。

需要注意的是，对于某些软件而言，自己配置软件确实是很无聊的事，因为它们并不需要个性化配置就能良好运作。但确实存在一些软件只有在配置后才会展示其强大的功能，例如，vim、xterm、zsh…

安装

Zsh 非常有名，可以在大多数 Linux 包管理器中找到。你可以使用 apt（适用于 Debian 系列）或 yum（适用于 RedHat 系列）或 pacman（适用于 Arch 系列）来轻松安装它。

成功安装后，输入 zsh 来确保 Zsh 已正确安装。然后你应该会看到一个简单的配置指南，用于创建 .zshrc 文件。

然后可以使用命令 chsh 将 Zsh 设置为默认 shell。有趣的是，当我使用 chsh -l 检查计算机上可用的 shell 时，我看到了两个 zsh 目录 — ‘/bin/zsh’ 和 ‘/usr/bin/zsh’。这不是 zsh 的问题，而是 Linux 系统的问题。通常一个新的软件二进制文件应该安装在 /usr/ 目录下。但由于有太多人在终端环境中使用 zsh，所以 Linux 系统在 /bin/ 下建立了一个链接。两者都可以正常使用。
