BONC虚拟机使用说明

Posted on 2018-05-22 Edited on 2019-08-18
Symbols count in article: 2.5k Reading time ≈ 2 mins.

一、加载镜像文件

学员在收到镜像文件之后，需要通过VMware Workstation创建虚拟机加载该镜像：

  1. 下载并安装VMware Workstation
  2. 解压镜像文件，将文件夹中的镜像配置文件BONCVM2.vmx鼠标拖到VMware Workstation界面左侧我的计算机
  3. 鼠标点击该虚拟机，选择开启此虚拟机，稍作等待后即可打开。
  4. 出现bonc login提示后，输入用户名：bonc，密码：bonc，回车键，即可成功以bonc用户登录虚拟机。

备注：虚拟机设置相关操作都在VMware Workstation上进行：
虚拟机设置：菜单栏->虚拟机->设置
关闭虚拟机：菜单栏->暂停键旁边的箭头->关闭客户机

二、检查镜像文件

学员在收到镜像文件之后，启动镜像之后请首先进行检查：

  1. 执行命令ifconfig，查看ip地址
  2. 执行命令hostname -i，查看ip地址
  3. 如果两个ip地址不一致，修改host文件，具体操作步骤如下：
    第一步：root模式下执行vi /etc/hosts（root用户的密码也是bonc）
    第二步：编辑文本内容：修改localhost对应的ip地址为ifconfig显示的ip地址
    第三步：修改之后输入:wq保存
    第四步：输入命令hostname -i查看ip地址是否和ifconfig地址是否一致

三、连接务器&配置ssh免密钥登陆

在加载虚拟机镜像之后，使用PuTTY连接虚拟机进行操作，WinSCP进行文件传输:

  1. 安装PuTTy
  2. 打开PuTTY，Host Name栏输入刚才虚拟机的ip地址（SSH端口22），
    默认的字体显示比较小，在 Appearance栏中设置字体为14号以上应该差不多了。设置完毕后回到Session>Saved Sessions中输入BONC或其他标记保存，这样下次登陆就不用重复设置了。

    设置完毕后点击open登陆。可以使用用户名密码的登陆方式，即在用户身份验证处输入用户名：bonc，密码：bonc；也可以使用ssh免密钥的方式登陆，参考：https://billystudio.github.io/2018/05/14/Using-SSH-in-PuTTY/
  3. 安装WinSCP，安装的时候会提示导入PuTTY中的连接信息，比较方便。

四、环境启动

上述步骤正确执行执行，进入环境启动：

  1. 启动hadoop

判别：输入jps查看进程数量是否正常，应该具有以下进程：
namenode
secondryNamenode
datanode
resourceManager
sourceManager
备注：上述五个进程缺一不可

  1. 启动hive

判别：正常进入sql黑窗口操作界面即可，jps进程应该有runjar进程

  1. 启动zookeeper

判别：zkServer.sh status 显示信息不报错
输入jps，判定进程是否有QuorumPeerMain，
备注：上述两个情况均正常，代表zookeeper启动成功

  1. 启动kafka

判别：执行kafka相关命令进行判定
输入jps，查看是否有kafka字样进程(无)
备注：下面是kafka简单操作命令，查看/创建/生产/消费
命令一：kafka-topics.sh —list —zookeeper localhost:2181 //查看所有topic
命令二：kafka-topics.sh —create —zookeeper localhost:2181 —replication-factor 1 —partitions 1 —topic test
命令三：kafka-console-producer.sh —broker-list localhost:9092 —topic test
在”>”之后随便输入消息
命令四：kafka-console-consumer.sh —zookeeper localhost:2181 —topic test —from-beginning

  1. 启动storm

判别：输入jps查看是否有以下进程
nimbus
supervisor

如果有代表storm的nimbus以及supervisor启动成功
备注：在浏览器中输入：hostname -i对应的ip地址
ip:18088访问storm 的ui监控界面
